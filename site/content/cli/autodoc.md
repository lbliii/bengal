**Command:** `bengal.autodoc`


ðŸ“š Generate comprehensive API documentation (Python + CLI + OpenAPI).

Automatically generates Python API, CLI, and OpenAPI docs based on
your bengal.toml configuration. Use --python-only, --cli-only, or
--openapi-only to generate specific types.

Documentation is generated as Markdown files in the configured output
directories.

## Usage

```bash
autodoc [OPTIONS] [ARGUMENTS]
```

## Options

### `source`


Source directory to document (can specify multiple)

**Type:** `path`





### `output`


Output directory for generated docs (default: from config or content/api)

**Type:** `path`





### `clean`


Clean output directory before generating

**Type:** `boolean`

**Default:** `False`




### `parallel`


Use parallel processing (default: enabled)

**Type:** `boolean`

**Default:** `True`




### `verbose`


Show detailed progress

**Type:** `boolean`

**Default:** `False`




### `stats`


Show performance statistics

**Type:** `boolean`

**Default:** `False`




### `config`


Path to config file (default: bengal.toml)

**Type:** `path`





### `python_only`


Only generate Python API docs (skip others)

**Type:** `boolean`

**Default:** `False`




### `cli_only`


Only generate CLI docs (skip others)

**Type:** `boolean`

**Default:** `False`




### `openapi_only`


Only generate OpenAPI docs (skip others)

**Type:** `boolean`

**Default:** `False`
