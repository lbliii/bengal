**Command:** `bengal.utils.autodoc`


ðŸ“š Generate comprehensive API documentation (Python + CLI).

Automatically generates both Python API docs and CLI docs based on
your bengal.toml configuration. Use --python-only or --cli-only to
generate specific types.

Documentation is generated as Markdown files in the configured output
directory (default: content/api for Python, content/cli for CLI).

## Usage

```bash
autodoc [OPTIONS] [ARGUMENTS]
```

## Options

### `source`


Source directory to document (can specify multiple)

**Type:** `path`





### `output`


Output directory for generated docs (default: from config or content/api)

**Type:** `path`





### `clean`


Clean output directory before generating

**Type:** `boolean`

**Default:** `False`




### `parallel`


Use parallel processing (default: enabled)

**Type:** `boolean`

**Default:** `True`




### `verbose`


Show detailed progress

**Type:** `boolean`

**Default:** `False`




### `stats`


Show performance statistics

**Type:** `boolean`

**Default:** `False`




### `config`


Path to config file (default: bengal.toml)

**Type:** `path`





### `python_only`


Only generate Python API docs (skip CLI docs)

**Type:** `boolean`

**Default:** `False`




### `cli_only`


Only generate CLI docs (skip Python API docs)

**Type:** `boolean`

**Default:** `False`
