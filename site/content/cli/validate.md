**Command:** `bengal.validate`


üîç Validate site health and content quality.

Runs health checks on your site to find errors, warnings, and issues.
By default, shows only problems (errors and warnings).

## Usage

```bash
validate [OPTIONS] [ARGUMENTS]
```

## Options

### `files`


Validate specific files (can be specified multiple times)

**Type:** `path`





### `changed`


Only validate changed files (requires incremental build cache)

**Type:** `boolean`

**Default:** `False`




### `watch`


Watch mode: validate on file changes (experimental)

**Type:** `boolean`

**Default:** `False`




### `profile`


Build profile to use (writer, theme-dev, developer)

**Type:** `choice`





### `verbose`


Show verbose output (all checks, not just problems)

**Type:** `boolean`

**Default:** `False`




### `suggestions`


Show quality suggestions (collapsed by default)

**Type:** `boolean`

**Default:** `False`




### `incremental`


Use incremental validation (only check changed files)

**Type:** `boolean`

**Default:** `False`




### `traceback`


Traceback verbosity: full | compact | minimal | off

**Type:** `choice`
