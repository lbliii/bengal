# Bengal 0.2.6

**Key additions:** smoother dev-server reloads, new AI-friendly site outputs, Content Signals enforcement, and a one-click Cursor MCP connect flow for docs sites.

---

## Highlights

### Faster, Safer Local Iteration

The dev server now supports a more reactive editing loop:

- **Content-only updates** can refresh the main content fragment without a full rebuild
- **Double-buffered output** reduces flashes of unstyled or partially written pages during rapid rebuilds
- **Local DX** is steadier with improved cache validation, reload decisions, and clearer stale-process / port handling

### AI and Agent Discovery Outputs

Bengal can now emit more machine-friendly site artifacts out of the box:

- **`llms.txt`**: a curated site overview following the [llms.txt spec](https://llmstxt.org/)
- **`agent.json`**: a hierarchical manifest for agent and tool navigation
- **`changelog.json`**: per-build page add/modify/remove diffs for incremental indexing workflows
- **Richer `index.json`**: per-page JSON can include navigation, last-modified data, content hashes, and heading-level chunks for RAG

### Content Signals and SEO Controls

Bengal now generates and documents stronger AI/search discovery controls:

- **`robots.txt`** with [Content Signals](https://contentsignals.org/) directives
- **`.well-known/content-signals.json`** policy manifest
- **Output enforcement** so pages excluded from `search`, `ai_input`, or `ai_train` do not get the corresponding machine-readable formats

### Cursor MCP Connect Links

Docs sites can now expose a **Connect to IDE** action that opens Cursor and offers a one-click MCP install flow for your hosted docs server.

Configure it with:

```toml
[connect_to_ide]
enabled = true
mcp_url = "https://docs.example.com/mcp"
server_name = "My Docs"
```

### Template and Authoring Fixes

- **Template context validation** catches missing Kida context variables before render
- **Frontmatter tag normalization** handles malformed `tags` input more safely
- **`list-table` fixes** improve option mapping and empty-cell rendering

---

## Docs Updated

This release also refreshes the docs and README around the new capabilities:

- **SEO & Discovery**: `robots.txt`, Content Signals, AI visibility, machine discovery
- **Output Formats**: `llms.txt`, `agent.json`, `changelog.json`, richer JSON exports
- **Connect to IDE**: Cursor MCP deeplink setup
- **Free-Threading**: runtime model and parallel rendering patterns

## Breaking Changes

None intended for typical sites. This release is primarily additive and focused on development workflow, discovery outputs, and documentation.
