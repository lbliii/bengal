The sidebar version switcher no longer drops section landing pages on versioned documentation sites. The switcher identified the active version through a value that is not always resolved when a section index page renders, which raised a template error and silently skipped every `_index` page (so `/docs/`, `/docs/get-started/`, and other section landing pages returned 404 on multi-version builds). It now matches the active version by the page's own version id, so section landing pages build and the current version is still marked in the menu.
