Fixed the `make gh-release` / `poe gh-release` mechanism: the GitHub release title was built from a greedy `grep '^name ='` that captured every `[[tool.towncrier.type]]` category name (`Added`, `Changed`, `Deprecated`, `Removed`, `Fixed`, `Security`), so the v0.4.1 title rendered as `bengal Added Changed Deprecated Removed Fixed Security 0.4.1`. A new `scripts/publish_github_release.py` now reads `[project].version` via `tomllib` and renders the title as `vX.Y.Z — <theme>`, deriving the theme from the release page's frontmatter `description:`. (#gh-release-title)
